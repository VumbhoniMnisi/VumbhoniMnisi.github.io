### Reflection on Coding in Markdown 

### Situation
I needed to create a GitHub portfolio that will showcase my CV and projects I worked/ working on.

### Task
I coded my CV using Markdown language, organizing my personal information, contact details, work history, education, skills, and references in a clean, readable format.

### Action
I learned GitHub Markdown language, with syntax such as:
- `##` and `###`for headings
- `<br>` for line breaks between contact details
- `*` for bullet points in job descriptions and skills

I structured my CV logically, starting with my name and profile, then contact information, followed by work history, educational qualifications, technical skills, and finally references.

### Result
I successfully created a professional, well-formatted CV using Markdown that displays correctly on GitHub Pages. I learned that Markdown is an easier way to format web content without needing complex HTML.

### Reflection on GitHub Pages

### Situation
I am required me to deploy my portfolio to GitHub Pages so it would be publicly accessible via a custom URL.

### Task
Take my repository containing the CV, video, projects I worked on and publish it as a live website.

## Action
I went to my repository settings, navigated to the "Pages" section, and set the source to the `main` branch. GitHub automatically generated the URL `https://vumbhonimnisi.github.io/`. I committed all my files (CV content and video), waited a few minutes for deployment, and tested the live link.

## Result
My portfolio is now live and accessible to anyone with the link. I learned that GitHub Pages provides free, reliable hosting for static websites, and it automatically updates whenever I push changes to my repository.
